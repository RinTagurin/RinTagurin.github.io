1 プログラミングの基本

プログラミングとは？

「プログラミング」とは、「プログラム」を作ることです。 では、「プログラム」とは何でしょうか？

「プログラム」は「コンピューターへの命令」のことです。

今やコンピューターは身の回りにあふれています。パソコンやスマートフォンだけではありません。毎日使うような、テレビ、炊飯器、冷蔵庫、洗濯機などにもコンピューターが内蔵されています。それらのコンピューターは「プログラム」という命令に従って動いています。私たちは毎日「プログラム」のおかげで快適な生活を送っているのです。

プログラムは何ができるの？

コンピューターにも得意なこと、不得意なことがあります。それらの特徴を知って、プログラミングができると何ができるようになるか、考えましょう。

得意なこと

  • 人間より速く計算できる
  • いつでも、いつまでも動くことができる

不得意なこと

  • 「感情」を扱うこと

・・・どうですか？プログラミングでコンピューターを操ることができるようになれば、これまで面倒だったことをコンピューターに任せることができるかもしれません。さらに、自分の力ではできなかったことをコンピューターに頼むことができるかもしれませんね。

「プログラミング」とは何なのか、イメージがつかめてきましたか？では、次にプログラムのしくみを見てみましょう。

プログラムのしくみ

プログラムの進み方には3つの大原則があります。

  • 順次

    「命令が書かれた順番通りに処理が進む」

  • 分岐

    「条件によって処理を枝分かれさせる」

  • 反復

    「一定の条件を満たすまで処理を繰り返す」

これは、プログラミングする上でとても大切な考え方です。どのプログラミング言語を使っても、この考え方は共通です。

・・・言語？不思議に思った方もいるかもしれませんね。実は、プログラムはコンピューターと友達になるための「言語」なのです。

プログラミング言語

世界には日本語、英語、ロシア語と、様々な言語があります。これは「人と人」がコミュニケーションをとるためのものですね。実は、「人とコンピューター」がコミュニケーションをとるための「言語」があって、これにも様々な種類があります。次に代表的なものを挙げます。聞いたことがある言語はありますか？

  • C言語

  • Ruby

プログラミング言語には100以上の種類があります。それぞれに特徴があって、その言語が「得意なこと」も存在します。

今回使用するのは「Python」というプログラミング言語です。「パイソン」と読みます。では、これから学ぶ「Python」について知りましょう。

Pythonとは？

「Python」は、1991年に開発されたプログラミング言語です。マイクロソフト社やアップル社でも利用される、世界中で人気のある言語です。今話題の「AI / 人工知能」の分野でも活躍しています。

なぜPythonなのか？

プログラミングを始めるにあたって「学びやすさ」は重要です。「Hello World!」と表示するプログラムを例にとって、先ほど挙げた「C言語」「Java」「Python」「Ruby」を比較してみましょう。

#include <stdio.h>

  • Java

print('Hello World!')

puts "Hello World!"

いかがでしょうか。「C言語」や「Java」では5行で書いていることが、「Python」や「Ruby」ではたったの1行で終わりです。いざプログラミングを始めるぞ、と手を出してみたものの、初歩的な段階で5行ものプログラムが出てきたらどうでしょう。これではきっと、勉強がなかなか進みませんね。その点「Python」や「Ruby」は親しみやすい言語だといえます。

では「Python」vs「Ruby」ですが、実はこれ、とてもいい勝負なのです。ここで両者のプロフィールを見てみましょう。

「Python」は世界中で使われている、すなわち、世界中でPythonが学ばれているということです。ですから、インターネットで検索した時の情報の量が豊富です。一方「Ruby」は日本発祥の言語ですから日本語での解説が充実しています。むむむ、互角だ。。。

2 Pythonの基本

文字を表示しよう

何かを始める時には「儀式」があるものです。プログラミングの世界でも「儀式」があります。Pythonを学び始める際に、決まって作られるプログラムがあるので紹介します。

print("Hello, World!")

「儀式」のプログラムといえば、長いものを思い浮かべるかもしれません。（卒業式とか・・・）でも、Pythonの儀式のプログラムはたった1行だけです。短いですが、これはPythonプログラミングの基礎の基礎の基礎です。

「 print() 」は、「括弧の中身を表示する」という命令です。この場合、中身は「 "Hello, World!" 」ですね。ここで重要なのが、記号「 " 」（ダブルクォーテーション）で表示したいものを囲むことです。これで囲んだものは「文字列」として扱われます。

文字の足し算

「文字列どうし」は足し算することができます。

print("Hello," + "World!")

文字列の足し算の方法は簡単です。文字列と文字列の間に記号「 + 」を挟むだけです。文字列の足し算をすることで、複数の文字列をまとめてひとつの文として表示することができます。

でも、今の時点では「文字列の足し算」の意味が感じられないかもしれません。わざわざ足し算を使わないでも、同じ「Hello,World!」を表示することができますもんね。しかし、この「文字列の足し算」は後々力を発揮します。ですから、今はこのまま覚えてください。

数を表示しよう

コンピューターは計算が得意なのですから、当然「数」を扱うことができます。その方法は、文字列の時とほとんど同じです。

print(1)

どうでしょうか、表示されましたか？では少しレベルアップして、計算をしてもらいましょう。

計算してもらおう

足し算、引き算、掛け算、割り算と、基本的な計算の仕方を覚えましょう。

print(2 + 3)
print(2 - 3)
print(3 * 5)
print(5 / 2)

足し算と引き算の時は、私たちが普段使う記号と同じものを使いますが、掛け算と割り算の時は違います。掛け算の記号は「 * 」、割り算の記号は「 / 」です。

計算の順番は、算数でのルールと同じです。次のプログラムで確かめることができます。

print(1 + 2 * 3)
print((1 + 2) * 3)

足し算より先に掛け算をして、括弧があればその中の計算を優先します。括弧の中に括弧が入り、少し見にくくなるので注意してください。エラーが出たら、括弧がきちんとペアになっているか確認しましょう。

数と文字列の違い

さて、ここは大変重要なポイントです。これまでに覚えた、数を表示するプログラムと、文字列を表示するプログラムを見比べてみましょう。

print(1)

print("Hello, World!")

違いが分かりますか？そう、ダブルクォーテーションです。上の「数」を表示するプログラムでは、ダブルクォーテーションは使われていません。一方の「文字列」を表示するプログラムでは、文字をダブルクオーテーションで囲むんでしたね。もし文字をダブルクォーテーションで囲まなかったら、エラーが出ます。

SyntaxError: invalid syntax

「構文エラー：無効な構文」という意味です。ダブルクォーテーションで囲ってあげないと、「文字」として扱われないので注意しましょう。

では逆に、数をダブルクォーテーションで囲んだらどうなるでしょう？

print(1)
print("1")

どうでしょうか。見た感じは結果は同じですね。しかし、見た目は同じでも中身は別です。ダブルクォーテーションで囲んでいない方は「数の1」であり、囲んだ方は「文字列の1」です。・・・すこし分かりにくいですね。では、次のようなプログラムを書いて実験してみましょう。

print(1 + 1)
print("1" + "1")

足し算のプログラムです。1行目では、「数の1」どうしを足していますから、結果は「数の2」が表示されます。そして2行目では、「文字列の1」どうしを足していて、結果は「文字列の11」です。これでなんとなく違いが分かったでしょうか？プログラミングの世界では、「数」と「文字列」は別物として扱います。数字をそのまま書くと「数」、ダブルクォーテーションで囲むと「文字列」になります。

ちなみに、「数」と「文字列」の足し算をさせるとどうなるか、見てみましょう。

print(1 + "1")

これを実行すると、エラーが出ます。

TypeError: unsupported operand type(s) for +: 'int' and 'str'

「型エラー：整数と文字列は足し算できないよ！」てな感じです。見た目は同じ、中身は別物。すこし厄介ですが、とても重要なポイントです。

「変数」って？

変数とは何でしょう？よく、変数は「値を入れる箱」と表現されます。実際にこの「変数」を使ったプログラムを書いてみましょう。

order = "うどん"
print(order)

このプログラムの1行目では、変数「order」に値「"うどん"」を代入しています。数学の「＝」と同じ記号を使いますが、Pythonの世界では少し違います。左側が変数、右側が代入する値です。左右を逆にすることはできません。

そして2行目で変数「order」の「中身」を表示しています。このとき、画面には「order」ではなく「うどん」と表示されるのがポイントです。

変数は便利！

たとえば、飲食店で注文をとるロボットがいるとしましょう。お客さんが「うどん」を注文したら、第一に「ご注文はうどんですね？」と確認をします。その次に厨房に向かって「うどん一丁！」と伝えます。これがロボットがするべき仕事です。では、このロボットのプログラムを書いてみましょう。まずは「変数」を使わないで書いてみてください。

print("ご注文はうどんですね？")
print("うどん一丁！")

確かに、このプログラムでもロボットは最低限の仕事をしてくれます。次に、別のお客さんが「そば」を注文した場合のプログラムを書いてみましょう。

print("ご注文はそばですね？")
print("そば一丁！")

きっと、「うどんを消して蕎麦と入力」という作業を2回行うことになるでしょう。このくらいならまだいいですが、次のお客さんが「ごぼう天うどんのミニカツ丼セット」を注文したらどうでしょうか。2回も長い商品名を入力するのはちょっと面倒ですよね。注文が長いので、厨房に間違った注文を伝えるミスも起こりやすくなります。

こんな時に力を発揮するのが「変数」なのです。変数を使ったプログラムに書き換えてみましょう。

order = "うどん"
print("ご注文は" + order + "ですね？")
print(order + "一丁！")

プログラムは3行に長くなりましたが、注文内容を変える時は1行目を書き換えるだけで済みます。

変数のイメージがつかめたでしょうか？これまでは飲食店のロボットの想定でしたが、今度はより現実的な例を見てみましょう。半径が2の円の面積を求めるプログラムを書いてみてください。「円の面積＝半径x半径x円周率」ですね。

  • 変数を使わない場合
print(2 * 2 * 3.14)
  • 変数を使う場合
r = 2
print(r * r * 3.14)

どうでしょうか？一回しか使わないプログラムであれば、わざわざ変数を使わないでもいいかもしれません。でも、世の中の円の半径は2だけではないですよね。そんなときも、変数を使えば1行目を書き換えるだけで済み、手間やミスも減ります。

ちなみに、変数の名前の付け方はとても重要です。大規模なプログラムを作るときは、何人か集まって協力しながらプログラミングをすることがあります。こんな時に変数の名前を意味がないもの（a,x,など）にすると、大変読みにくいプログラムになってしまいます。ですから、変数の名前はできる限り内容がすぐ分かるようなものにするのが大切です。仲間のため、あるいは将来の自分のためにも、変数の名前はよく考えて名付けましょう。

「input()」を使ってみよう

Pythonでは出力だけでなく、入力することもできます。「input()」は文字列の入力を行う命令です。次のプログラムで確認してみましょう。

a = input("入力してください")
print("あなたは「" + a + "」と入力しました。")

プログラムを利用する段階で「入力」ができると、それに従った動きをさせることができます。次の章で、処理の流れを変える方法を学びましょう。

3 処理の流れを変える「分岐」

これまでのプログラムは、決まった処理を順番通りに進んでいく、とても単純なプログラムでした。では次に、「OOだったら、XXする。」この表現を覚えましょう。プログラムの三大原則のひとつ「分岐」です。条件によって処理を変えることができます。

条件って？

「条件」とは、「OOだったら」の部分です。「変数xが10より大きかったら」とか、「aが1だったら」とかです。これらのことをPythonではどのように表すのでしょうか？

数の大きさを比較

数の大きさを比較するときには、主に3つの記号を使います。

  • 「<」
  • 「>」
  • 「==」 ・・・2文字です！注意！

これらの記号を使うことで、「数の大きさ」を比べることができますが、Pythonが返すことができる答えは2種類しかありません。実際にプログラムを書いてみましょう。

print(3 < 2)
print(3 == 2)
print(3 == 3)
print(3 > 2)

実行します。

False
False
True
True

どうでしょう。返ってくる答えの規則が分かりますか？

「True」は「正しい」、「False」は「正しくない」ことを意味します。不等号や「==」を使うことによって、その式が「正しい」か「正しくない」のかを知ることができます。

これで、「OOだったら」の条件の部分を表現することができました。次に、実際に処理を変える方法を見てみましょう。

「もし」OOだったらXXする

「もし」は、処理の分岐の基本です。（個人的なイメージは、物流倉庫のベルトコンベヤーの上の荷物が宛て先によって振り分けられていく感じです。）「もしも」の話をするときは英語で「if」を使いますよね。これはPythonでも同じです。

if 「条件」: #「:」コロンを忘れずに！
    「条件を満たしたときの処理」
    （左の余白も重要です！）
else:
    「条件を満たさなかったときの処理」

実際に書いてみましょう。年齢を尋ねて、成人かどうかを判断するプログラムを作ってみましょう。「年齢」は英語で「age」です。

age = input("あなたの年齢を入力してください。")
if int(age) < 20:
    print("あなたは未成年です。")
else:
    print("あなたは成人です。")

1行目で、変数「age」に入力された年齢を代入します。（このとき、input()で入力された年齢は「文字列」です。文字列を「int()」のかっこで囲むことで、整数に変換して扱うことができます。）2行目で、年齢が20より小さいかどうかを判断します。「age」に14が代入されていたとすると条件「14 < 20」が成り立つので「True」となり、3行目に進むことができます。例えば「age」が24の場合は、「24 < 20」は成り立たないので「False」となり、3行目を飛ばして「else」の処理に入ります。

4 処理の流れを変える「反復」

次は、プログラムの三大原則の三つ目「反復」です。「反復」を使うことができるようになれば、面倒な単純作業をコンピューターに任せることができます。さらに、人間にはできないような規模の大きな処理を頼むこともできます。「反復」はとても便利な考え方です。

「for」繰り返し

繰り返しにはいくつか種類がありますが、ここでは最も単純な繰り返しを使えるようになりましょう。「同じ処理を」何回も繰り返すことができます。その文法はこちら。

for i in range(n): # nは繰り返す回数
    「繰り返したい処理」
（↑ここの余白が重要です！）

これだけで、同じ処理を何回も繰り返すことができます。（本来はもっと深い意味があるのですが、それはまた次のステップで。）

実際のプログラムで確認してみましょう。

for i in range(5):
    print("Hello, World!")

どうでしょう。同じ文が5行表示されましたね。

意味を持つ「空白」

Pythonでは「空白」が重要な意味をもっていて、「分岐」や「反復」を表すときに登場します。意味のある「空白」のことを「字下げ」や「インデント」と言います。通常、半角スペースが4つでひとつの「字下げ」です。「字下げ」した部分を「ブロック」といい、処理のまとまりを表します。

「分岐」を思い出して確認しましょう。

if 「条件」:
    「処理A」
else:
    「処理B」

では次に、「反復」の場合も。

for i in range(n):
    「処理」

「字下げ」のミスをすると構造が変わってしまい、予期しない動きをしたり、エラーが出たりします。見た目は「空白」でも、Pythonの世界では重要な意味を持っているので、注意しましょう。

「for」繰り返しは便利！

先ほどの例では、「for」を使った同じ処理の繰り返しで、「Hello, World!」を5回表示しました。でも、これでは「for」繰り返しのパワーを使いこなしているとはいえません。「変数」と組み合わせることによって、さらなるパワーを発揮させることができます。少し複雑になりますが、ゆっくり考えて理解しましょう。まずは、次のプログラムを見てください。

n = 1
for i in range(5):
    print(n)

実行する前に、どんな動きをするか予想してみましょう。予想がついたら、実行してみてください。

1
1
1
1
1

予想は当たりましたか？「print(n)」という同じ処理を5回繰り返す、ということで変数「n」に代入されていた数「1」が5回表示されましたね。

ではでは次に、繰り返しのブロックの中に「n = n + 1」の1行を書き加えてください。プログラム全体はこうなります。

n = 1
for i in range(5):
    print(n)
    n = n + 1 # 書き加えた部分

さて、「n = n + 1」について詳しく考えてみましょう。数学の感覚で見ると、この式には違和感がありますね。「左辺と、左辺に1を足した数は等しい」・・・むむむ？成り立つの？。

では、見方のスイッチを「プログラミング」に切り替えてください。すると、記号「=」は、「等しい」という意味から「代入」という意味に切り替わります。プログラミングの見方で、もう一度見てみましょう。「変数nに、『変数nに1足したもの』を代入する」と読めますか？ 例を挙げると、変数nに「1」が代入されているときに「n = n + 1」の処理が入ると、変数nには新たに「2」が代入されます。このプログラムの場合は、それがforループで繰り返されます。実際に実行して確かめましょう。

1
2
3
4
5

この「n = n + 1」という処理は物事をカウントする際などでよく使う、便利な表現です。

では演習として、「変数」「反復」に加えて、「分岐」も取り入れたプログラムをみてみましょう。テーマは「数当てゲーム」です。ルールは簡単。挑戦者が数を入力すると、正解より大きいか小さいかが返ってきます。限られた回数の中で正解にたどり着けるか？というゲームです。プログラムはこちら。

atari = 5
n = 5 # n回までチャレンジできる
r = 0 # カウント用
for i in range(n):
    r = r + 1
    print(r, "回目の予想 残り", n-r+1, "回です。")
    yosou = input("数を入力してください >>> ")
    yosou = int(yosou) # 文字列を整数に変換
    if yosou < atari:
        print("もっと大きいです。\n")
    elif yosou > atari:
        print("もっと小さいです。\n")
    else:
        print("あたりです！")
        break # forループを抜け出す
print("正解は", atari, "でした。")

どうでしょうか？プログラミング特有の考え方に慣れてきましたか？「数当てゲーム」なのに、答えが常に同じだったらあまり楽しくないですよね。ということで、次は「ランダム」を扱います。ランダムな要素が入ってくると、意図しない処理の進み方になったり、エラーが出たりしやすくなります。これまでの「基本」アイテムの使い方をしっかりと身につけましょう。

5 「ランダム」を扱う

「random」モジュールのインポート

これからは、「ランダム」を扱います。しかし「ランダム」はPythonの初期装備には含まれていません。でも大丈夫、Pythonには「ランダム」の世界を扱うことができるようになる魔法の一文があります。

import random

この一行でにふりがなを付けるとすると「『ランダム』モジュールをインポートする」といったところでしょうか。モジュールは「道具箱」のようなイメージです。「import random」でランダムに関するアイテムを、「import time」で時間に関するアイテムを使うことができます。

アイテムを使ってみよう

では「ランダム」モジュールのアイテムを見てみましょう。モジュールのインポートで追加装備したアイテムを使うときには、「モジュール.アイテム」と書きます。

import random
a = random.random()
print(a)

このプログラムを実行してみてください。「0.xxxxxxxx」と、小数が表示されましたか？この「random.random()」というアイテムは、「0から1」の間の数をランダムに返すアイテムです。繰り返して、ランダムであることを確認してみましょう。

import random

for i in range(5):
    a = random.random()
    print(a)

実行してみます。

0.13114344772814235
0.4670234470648774
0.7762781564874052
0.9061266635955899
0.609032316190615

毎回ランダムな数が出力されていますね。

「random.random()」の利用

では、「random.random()」を、以前作った「数当てゲーム」に組み込んでみましょう。今回は、「0」から「9」までの整数をランダムに選んで「正解」の数とする仕様にします。「random.random()」から1桁の整数をランダムに取り出すにはどうすればいいでしょうか？考えてみてください。

おそらくもっとも簡単なのは、「小数点以下第1位の数字」を利用することです。ランダムな小数を10倍して小数点以下を切り捨てれば、ランダムな1桁の整数を取り出すことができますね。これは次の処理で実現できます。

import random
a = int(random.random() * 10)
print(a)

これも繰り返して、予想通りの動きをするか確かめてみましょう。

import random

for i in range(5):
    a = int(random.random() * 10)
    print(a)

次は、実行結果の例です。

いい感じですね！ではこれを、以前の「数当てゲーム」のプログラムに組み込んでみましょう。

import random # モジュールのインポートも忘れずに！

atari = int(random.random() * 10) # ここを書き換えました
n = 5
r = 0
for i in range(n):
    r = r + 1
    print(r, "回目の予想 残り", n-r+1, "回です。")
    yosou = input("数を入力してください >>> ")
    yosou = int(yosou)
    if yosou < atari:
        print("もっと大きいです。\n")
    elif yosou > atari:
        print("もっと小さいです。\n")
    else:
        print("あたりです！")
        break
print("正解は", atari, "でした。")

これで、正解の数は毎回異なります。実行して遊んでみてください。
