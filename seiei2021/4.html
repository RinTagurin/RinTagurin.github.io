4 処理の流れを変える「反復」

次は、プログラムの三大原則の三つ目「反復」です。「反復」を使うことができるようになれば、面倒な単純作業をコンピューターに任せることができます。さらに、人間にはできないような規模の大きな処理を頼むこともできます。「反復」はとても便利な考え方です。

「for」繰り返し

繰り返しにはいくつか種類がありますが、ここでは最も単純な繰り返しを使えるようになりましょう。「同じ処理を」何回も繰り返すことができます。その文法はこちら。

for i in range(n): # nは繰り返す回数
    「繰り返したい処理」
（↑ここの余白が重要です！）

これだけで、同じ処理を何回も繰り返すことができます。（本来はもっと深い意味があるのですが、それはまた次のステップで。）

実際のプログラムで確認してみましょう。

for i in range(5):
    print("Hello, World!")

どうでしょう。同じ文が5行表示されましたね。

意味を持つ「空白」

Pythonでは「空白」が重要な意味をもっていて、「分岐」や「反復」を表すときに登場します。意味のある「空白」のことを「字下げ」や「インデント」と言います。通常、半角スペースが4つでひとつの「字下げ」です。「字下げ」した部分を「ブロック」といい、処理のまとまりを表します。

「分岐」を思い出して確認しましょう。

if 「条件」:
    「処理A」
else:
    「処理B」

では次に、「反復」の場合も。

for i in range(n):
    「処理」

「字下げ」のミスをすると構造が変わってしまい、予期しない動きをしたり、エラーが出たりします。見た目は「空白」でも、Pythonの世界では重要な意味を持っているので、注意しましょう。

「for」繰り返しは便利！

先ほどの例では、「for」を使った同じ処理の繰り返しで、「Hello, World!」を5回表示しました。でも、これでは「for」繰り返しのパワーを使いこなしているとはいえません。「変数」と組み合わせることによって、さらなるパワーを発揮させることができます。少し複雑になりますが、ゆっくり考えて理解しましょう。まずは、次のプログラムを見てください。

n = 1
for i in range(5):
    print(n)

実行する前に、どんな動きをするか予想してみましょう。予想がついたら、実行してみてください。

1
1
1
1
1

予想は当たりましたか？「print(n)」という同じ処理を5回繰り返す、ということで変数「n」に代入されていた数「1」が5回表示されましたね。

ではでは次に、繰り返しのブロックの中に「n = n + 1」の1行を書き加えてください。プログラム全体はこうなります。

n = 1
for i in range(5):
    print(n)
    n = n + 1 # 書き加えた部分

さて、「n = n + 1」について詳しく考えてみましょう。数学の感覚で見ると、この式には違和感がありますね。「左辺と、左辺に1を足した数は等しい」・・・むむむ？成り立つの？。

では、見方のスイッチを「プログラミング」に切り替えてください。すると、記号「=」は、「等しい」という意味から「代入」という意味に切り替わります。プログラミングの見方で、もう一度見てみましょう。「変数nに、『変数nに1足したもの』を代入する」と読めますか？ 例を挙げると、変数nに「1」が代入されているときに「n = n + 1」の処理が入ると、変数nには新たに「2」が代入されます。このプログラムの場合は、それがforループで繰り返されます。実際に実行して確かめましょう。

1
2
3
4
5

この「n = n + 1」という処理は物事をカウントする際などでよく使う、便利な表現です。

では演習として、「変数」「反復」に加えて、「分岐」も取り入れたプログラムをみてみましょう。テーマは「数当てゲーム」です。ルールは簡単。挑戦者が数を入力すると、正解より大きいか小さいかが返ってきます。限られた回数の中で正解にたどり着けるか？というゲームです。プログラムはこちら。

atari = 5
n = 5 # n回までチャレンジできる
r = 0 # カウント用
for i in range(n):
    r = r + 1
    print(r, "回目の予想 残り", n-r+1, "回です。")
    yosou = input("数を入力してください >>> ")
    yosou = int(yosou) # 文字列を整数に変換
    if yosou < atari:
        print("もっと大きいです。\n")
    elif yosou > atari:
        print("もっと小さいです。\n")
    else:
        print("あたりです！")
        break # forループを抜け出す
print("正解は", atari, "でした。")

どうでしょうか？プログラミング特有の考え方に慣れてきましたか？「数当てゲーム」なのに、答えが常に同じだったらあまり楽しくないですよね。ということで、次は「ランダム」を扱います。ランダムな要素が入ってくると、意図しない処理の進み方になったり、エラーが出たりしやすくなります。これまでの「基本」アイテムの使い方をしっかりと身につけましょう。
