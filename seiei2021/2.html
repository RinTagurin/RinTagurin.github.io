2 Pythonの基本

文字を表示しよう

何かを始める時には「儀式」があるものです。プログラミングの世界でも「儀式」があります。Pythonを学び始める際に、決まって作られるプログラムがあるので紹介します。

print("Hello, World!")

「儀式」のプログラムといえば、長いものを思い浮かべるかもしれません。（卒業式とか・・・）でも、Pythonの儀式のプログラムはたった1行だけです。短いですが、これはPythonプログラミングの基礎の基礎の基礎です。

「 print() 」は、「括弧の中身を表示する」という命令です。この場合、中身は「 "Hello, World!" 」ですね。ここで重要なのが、記号「 " 」（ダブルクォーテーション）で表示したいものを囲むことです。これで囲んだものは「文字列」として扱われます。

文字の足し算

「文字列どうし」は足し算することができます。

print("Hello," + "World!")

文字列の足し算の方法は簡単です。文字列と文字列の間に記号「 + 」を挟むだけです。文字列の足し算をすることで、複数の文字列をまとめてひとつの文として表示することができます。

でも、今の時点では「文字列の足し算」の意味が感じられないかもしれません。わざわざ足し算を使わないでも、同じ「Hello,World!」を表示することができますもんね。しかし、この「文字列の足し算」は後々力を発揮します。ですから、今はこのまま覚えてください。

数を表示しよう

コンピューターは計算が得意なのですから、当然「数」を扱うことができます。その方法は、文字列の時とほとんど同じです。

print(1)

どうでしょうか、表示されましたか？では少しレベルアップして、計算をしてもらいましょう。

計算してもらおう

足し算、引き算、掛け算、割り算と、基本的な計算の仕方を覚えましょう。

print(2 + 3)
print(2 - 3)
print(3 * 5)
print(5 / 2)

足し算と引き算の時は、私たちが普段使う記号と同じものを使いますが、掛け算と割り算の時は違います。掛け算の記号は「 * 」、割り算の記号は「 / 」です。

計算の順番は、算数でのルールと同じです。次のプログラムで確かめることができます。

print(1 + 2 * 3)
print((1 + 2) * 3)

足し算より先に掛け算をして、括弧があればその中の計算を優先します。括弧の中に括弧が入り、少し見にくくなるので注意してください。エラーが出たら、括弧がきちんとペアになっているか確認しましょう。

数と文字列の違い

さて、ここは大変重要なポイントです。これまでに覚えた、数を表示するプログラムと、文字列を表示するプログラムを見比べてみましょう。

print(1)

print("Hello, World!")

違いが分かりますか？そう、ダブルクォーテーションです。上の「数」を表示するプログラムでは、ダブルクォーテーションは使われていません。一方の「文字列」を表示するプログラムでは、文字をダブルクオーテーションで囲むんでしたね。もし文字をダブルクォーテーションで囲まなかったら、エラーが出ます。

SyntaxError: invalid syntax

「構文エラー：無効な構文」という意味です。ダブルクォーテーションで囲ってあげないと、「文字」として扱われないので注意しましょう。

では逆に、数をダブルクォーテーションで囲んだらどうなるでしょう？

print(1)
print("1")

どうでしょうか。見た感じは結果は同じですね。しかし、見た目は同じでも中身は別です。ダブルクォーテーションで囲んでいない方は「数の1」であり、囲んだ方は「文字列の1」です。・・・すこし分かりにくいですね。では、次のようなプログラムを書いて実験してみましょう。

print(1 + 1)
print("1" + "1")

足し算のプログラムです。1行目では、「数の1」どうしを足していますから、結果は「数の2」が表示されます。そして2行目では、「文字列の1」どうしを足していて、結果は「文字列の11」です。これでなんとなく違いが分かったでしょうか？プログラミングの世界では、「数」と「文字列」は別物として扱います。数字をそのまま書くと「数」、ダブルクォーテーションで囲むと「文字列」になります。

ちなみに、「数」と「文字列」の足し算をさせるとどうなるか、見てみましょう。

print(1 + "1")

これを実行すると、エラーが出ます。

TypeError: unsupported operand type(s) for +: 'int' and 'str'

「型エラー：整数と文字列は足し算できないよ！」てな感じです。見た目は同じ、中身は別物。すこし厄介ですが、とても重要なポイントです。

「変数」って？

変数とは何でしょう？よく、変数は「値を入れる箱」と表現されます。実際にこの「変数」を使ったプログラムを書いてみましょう。

order = "うどん"
print(order)

このプログラムの1行目では、変数「order」に値「"うどん"」を代入しています。数学の「＝」と同じ記号を使いますが、Pythonの世界では少し違います。左側が変数、右側が代入する値です。左右を逆にすることはできません。

そして2行目で変数「order」の「中身」を表示しています。このとき、画面には「order」ではなく「うどん」と表示されるのがポイントです。

変数は便利！

たとえば、飲食店で注文をとるロボットがいるとしましょう。お客さんが「うどん」を注文したら、第一に「ご注文はうどんですね？」と確認をします。その次に厨房に向かって「うどん一丁！」と伝えます。これがロボットがするべき仕事です。では、このロボットのプログラムを書いてみましょう。まずは「変数」を使わないで書いてみてください。

print("ご注文はうどんですね？")
print("うどん一丁！")

確かに、このプログラムでもロボットは最低限の仕事をしてくれます。次に、別のお客さんが「そば」を注文した場合のプログラムを書いてみましょう。

print("ご注文はそばですね？")
print("そば一丁！")

きっと、「うどんを消して蕎麦と入力」という作業を2回行うことになるでしょう。このくらいならまだいいですが、次のお客さんが「ごぼう天うどんのミニカツ丼セット」を注文したらどうでしょうか。2回も長い商品名を入力するのはちょっと面倒ですよね。注文が長いので、厨房に間違った注文を伝えるミスも起こりやすくなります。

こんな時に力を発揮するのが「変数」なのです。変数を使ったプログラムに書き換えてみましょう。

order = "うどん"
print("ご注文は" + order + "ですね？")
print(order + "一丁！")

プログラムは3行に長くなりましたが、注文内容を変える時は1行目を書き換えるだけで済みます。

変数のイメージがつかめたでしょうか？これまでは飲食店のロボットの想定でしたが、今度はより現実的な例を見てみましょう。半径が2の円の面積を求めるプログラムを書いてみてください。「円の面積＝半径x半径x円周率」ですね。

  • 変数を使わない場合
print(2 * 2 * 3.14)
  • 変数を使う場合
r = 2
print(r * r * 3.14)

どうでしょうか？一回しか使わないプログラムであれば、わざわざ変数を使わないでもいいかもしれません。でも、世の中の円の半径は2だけではないですよね。そんなときも、変数を使えば1行目を書き換えるだけで済み、手間やミスも減ります。

ちなみに、変数の名前の付け方はとても重要です。大規模なプログラムを作るときは、何人か集まって協力しながらプログラミングをすることがあります。こんな時に変数の名前を意味がないもの（a,x,など）にすると、大変読みにくいプログラムになってしまいます。ですから、変数の名前はできる限り内容がすぐ分かるようなものにするのが大切です。仲間のため、あるいは将来の自分のためにも、変数の名前はよく考えて名付けましょう。

「input()」を使ってみよう

Pythonでは出力だけでなく、入力することもできます。「input()」は文字列の入力を行う命令です。次のプログラムで確認してみましょう。

a = input("入力してください")
print("あなたは「" + a + "」と入力しました。")

プログラムを利用する段階で「入力」ができると、それに従った動きをさせることができます。次の章で、処理の流れを変える方法を学びましょう。
